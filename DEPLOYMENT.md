# Deploying Lumina

Two pieces to deploy: the **backend** (FastAPI) and the **frontend** (React).
The database is already on MongoDB Atlas (cloud), so nothing to do there.

Recommended hosts (both have free tiers and deploy from GitHub):

- **Backend ‚Üí [Render](https://render.com)**
- **Frontend ‚Üí [Vercel](https://vercel.com)**

---

## 1. Deploy the backend (Render)

1. Push this repo to GitHub (already done).
2. On Render: **New ‚Üí Blueprint**, select this repo. It reads `render.yaml`
   and creates the `lumina-backend` service (root dir `backend/`).
3. Fill in the secret env vars when prompted (values from your local
   `backend/.env` ‚Äî see `backend/.env.example` for the full list):
   - `MONGO_URL`
   - `GROQ_API_KEY`
   - `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` (leave blank to disable Google login)
   - `FRONTEND_URL` and `CORS_ORIGINS` ‚Äî set **after** step 2 below, to your Vercel URL
   - `JWT_SECRET` is auto-generated by Render
4. Deploy. When it's live you'll get a URL like
   `https://lumina-backend.onrender.com`. Confirm it works: visiting
   `https://lumina-backend.onrender.com/api` should return `{"status":"ok"}`.

> **Free tier note:** Render spins the service down after inactivity, so the
> first request after idle takes ~30‚Äì50s to wake. Fine for a demo.

---

## 2. Deploy the frontend (Vercel)

1. On Vercel: **Add New ‚Üí Project**, select this repo.
2. Set **Root Directory** to `frontend`.
3. Framework preset: **Create React App** (Vercel auto-detects it).
4. Add an environment variable **before** the first build:
   - `REACT_APP_BACKEND_URL` = your Render backend URL (e.g.
     `https://lumina-backend.onrender.com`) ‚Äî no trailing slash, no `/api`.
5. Deploy. You'll get a URL like `https://luminaai.vercel.app`.

> The backend URL is baked in at **build time**, so if you change it later you
> must **redeploy** the frontend (Vercel ‚Üí Deployments ‚Üí Redeploy).
> `vercel.json` already handles SPA routing so deep links like `/dashboard`
> don't 404.

---

## 3. Wire the two together

1. Back on Render, set on the backend service:
   - `FRONTEND_URL` = your Vercel URL (`https://luminaai.vercel.app`)
   - `CORS_ORIGINS` = same Vercel URL
   Save ‚Üí it redeploys.
2. If using Google login, add the production callback in
   [Google Cloud Console](https://console.cloud.google.com) ‚Üí
   Credentials ‚Üí your OAuth client ‚Üí Authorized redirect URIs:
   `https://lumina-backend.onrender.com/api/auth/google/callback`

Done. Every `git push` to `main` now auto-redeploys both.

---

## Known limitation: uploaded files

R√©sum√©s and recordings save to a local `uploads/` folder. Render's disk is
**ephemeral** ‚Äî those files are wiped on each redeploy/restart. Interview data,
profiles, and streaks are safe (they live in MongoDB); only the raw uploaded
files are affected. For permanent file storage, move `put_object`/`get_object`
in `backend/core.py` to a cloud bucket (S3, Cloudflare R2, etc.).
